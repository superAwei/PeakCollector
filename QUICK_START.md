## 1. 啟動專案

專案已在 `http://localhost:3000` 運行中！

如果需要重新啟動：
```bash
cd /Users/awei/peak-collector
npm run dev
```

## 2. 測試 GPX 上傳功能

### 方法一：使用內建測試檔案

專案包含兩個測試 GPX 檔案，位於 `public` 目錄：

1. **test-yushan.gpx** - 玉山主峰軌跡
   - 包含玉山主峰的 GPS 軌跡點
   - 驗證後會點亮「玉山」徽章

2. **test-multiple.gpx** - 玉山群峰軌跡
   - 包含玉山主峰、玉山東峰、玉山北峰
   - 驗證後會同時點亮 3 座百岳徽章

### 測試步驟：

1. 開啟瀏覽器訪問 http://localhost:3000
2. 在「上傳 GPX 軌跡」區域，可以：
   - **拖放檔案**：將 `public/test-yushan.gpx` 拖到上傳區
   - **點擊選擇**：點擊上傳區，選擇測試檔案

3. 觀察驗證結果：
   - ✅ 成功：顯示綠色訊息，列出完成的百岳
   - 📍 對應的徽章會從灰色變成彩色
   - 🆕 新完成的徽章顯示 "NEW!" 標記
   - 📊 進度條和統計數據更新

## 3. 測試各項功能

## 6. 自訂測試

### 創建自己的測試 GPX

可以使用以下座標點測試其他百岳：

| 百岳 | 緯度 | 經度 | 海拔 |
|------|------|------|------|
| 雪山 | 24.3895 | 121.2342 | 3886m |
| 秀姑巒山 | 23.4561 | 121.0192 | 3825m |
| 南湖大山 | 24.3647 | 121.4322 | 3742m |

GPX 格式範例：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<gpx version="1.1" xmlns="http://www.topografix.com/GPX/1/1">
  <trk>
    <name>測試軌跡</name>
    <trkseg>
      <trkpt lat="24.3895" lon="121.2342">
        <ele>3886</ele>
      </trkpt>
    </trkseg>
  </trk>
</gpx>
```

## 7. 故障排除

### 問題：上傳後顯示「沒有經過任何百岳」
- 檢查 GPX 檔案格式是否正確
- 確認軌跡點座標是否在山頂 100 公尺內
- 使用內建測試檔案驗證功能正常

### 問題：進度沒有儲存
- 檢查瀏覽器 localStorage 是否啟用
- 不要使用無痕模式
- 檢查瀏覽器控制台是否有錯誤訊息

### 問題：徽章沒有點亮
- 重新整理頁面
- 檢查瀏覽器控制台錯誤
- 清除進度後重新上傳
